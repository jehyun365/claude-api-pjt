# PRD_01: 냉장고 이미지 인식

## 개요
사용자가 냉장고 내부 사진을 업로드하면, Vision 모델을 통해 사진 속 식재료 목록을 인식하는 기능.
전체 파이프라인(이미지 인식 → 레시피 생성 → 프로필/저장)의 1단계이며, 이 단계의 출력은 2단계(PRD_02)의 입력으로 사용된다.

## 목표
- 사용자가 냉장고 사진을 업로드하면, 사진에 포함된 식재료명 목록을 텍스트로 반환한다.
- 별도 회원 기능 없이 단일 요청-응답으로 동작하는 최소 기능(MVP)으로 시작한다.

## 범위
### 포함
- 이미지 업로드 UI (파일 선택 또는 드래그앤드롭, 1장)
- 업로드된 이미지를 base64로 인코딩하여 백엔드로 전달
- 백엔드에서 OpenRouter API(`nvidia/nemotron-nano-12b-v2-vl:free`)로 이미지 인식 요청
- 인식된 식재료 목록을 화면에 표시
- 인식 실패/에러 처리 (재시도 안내 포함)

### 제외 (다음 단계 또는 이후로 이연)
- 레시피 생성 (PRD_02)
- 사용자 계정/저장 (PRD_03)
- 다중 이미지 업로드
- 식재료 수량/유통기한 추정

## 사용자 흐름
1. 사용자가 웹 페이지에서 냉장고 사진을 업로드한다.
2. "인식하기" 버튼 클릭 시 이미지가 서버로 전송된다.
3. 서버가 OpenRouter Vision 모델에 이미지를 전달하고 식재료 목록을 요청한다.
4. 인식 결과(식재료명 리스트)를 화면에 표시한다.

## 기술 요구사항
- **모델**: `nvidia/nemotron-nano-12b-v2-vl:free` (OpenRouter, text+image → text)
- **인증**: `.env`의 `OPENROUTER_API_KEY` 사용, 서버 사이드에서만 키 사용 (클라이언트에 노출 금지)
- **이미지 전달 방식**: base64 data URI (`image_url.url = data:image/jpeg;base64,...`)
  - 외부 URL 방식은 provider가 핫링크 차단된 이미지를 가져오지 못하는 문제가 확인된 바 있어 사용하지 않음
- **프롬프트 설계**: 모델에 "이미지 속 식재료명만 목록으로 응답" 형태의 지시 필요 (자유 서술형 응답 방지)
  - 출력 형식을 JSON 배열로 강제하는 프롬프트 설계 권장 (예: `["계란", "우유", "당근"]`)
- **입력 제약**: 이미지 크기 제한(예: 5MB 이하), 지원 포맷(jpg/png)
- **에러 처리**: OpenRouter free 모델 특성상 502/504 발생 가능 → 최소 1회 자동 재시도 로직 필요

## API 인터페이스 (초안)
```
POST /api/recognize
Content-Type: application/json
Body: { "image": "data:image/jpeg;base64,..." }

Response 200:
{ "ingredients": ["계란", "우유", "당근", "양파"] }

Response 4xx/5xx:
{ "error": "메시지" }
```

## 성공 기준
- 실제 냉장고 사진 5장 이상으로 테스트 시, 육안으로 확인 가능한 주요 식재료(3개 이상)를 놓치지 않고 인식
- 잘못된 파일(비-이미지) 업로드 시 명확한 에러 안내
- API 실패 시에도 화면이 깨지지 않고 재시도 안내 표시

## 미결 사항
- 인식 결과가 부정확할 경우 사용자가 직접 수정/추가할 수 있는 UI 필요 여부
- 프롬프트에서 식재료 인식 정확도를 어떻게 검증할지 (수동 테스트 vs 자동화)
