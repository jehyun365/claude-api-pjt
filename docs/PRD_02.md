# PRD_02: 레시피 생성

## 개요
PRD_01에서 인식된 식재료 목록을 입력으로 받아, 언어 모델을 통해 해당 재료로 만들 수 있는 레시피를 생성하는 기능.
전체 파이프라인의 2단계이며, 이 단계의 출력은 3단계(PRD_03)에서 저장 대상이 된다.

## 목표
- 식재료 목록을 받아 실제로 만들 수 있는 레시피(요리명, 재료, 조리 순서)를 생성한다.
- 냉장고에 없는 재료가 필요한 경우 이를 별도로 구분해 안내한다.

> **범위 변경**: 실제 테스트 결과, 한 번에 레시피 2개 이상을 요청하면 두 번째 레시피부터
> 응답 중간에 JSON 구조를 깨뜨리는 의미 없는 텍스트가 삽입되는 문제가 확인됨(free 모델의 장문 생성 한계).
> 1개만 요청했을 때는 안정적으로 정상 JSON이 생성되어, MVP는 **레시피 1개 추천**으로 범위를 좁힘.

## 범위
### 포함
- PRD_01의 식재료 목록(JSON)을 입력받아 레시피 생성 요청
- `openai/gpt-oss-20b:free` 모델을 텍스트 전용으로 호출 (이미지 입력 없음)
- 레시피 1개 추천, 요리명/필요 재료/조리 순서/예상 조리 시간 포함
- 냉장고 재료만으로 부족한 경우, 추가로 필요한 재료를 별도 표시
- 생성 결과 화면 표시

### 제외
- 사용자 계정/레시피 저장 (PRD_03)
- 영양 정보, 칼로리 계산
- 레시피 이미지 생성

## 사용자 흐름
1. PRD_01 단계에서 인식된 식재료 목록이 화면에 표시된 상태에서, 사용자가 필요시 목록을 수정한다.
2. "레시피 추천" 버튼 클릭 시 식재료 목록이 서버로 전송된다.
3. 서버가 텍스트 모델에 재료 목록을 전달하고 레시피를 요청한다.
4. 추천 레시피(요리명, 재료, 조리 순서, 부족한 재료)를 화면에 표시한다.

## 기술 요구사항
- **모델**: `openai/gpt-oss-20b:free`
  - 초기에는 PRD_01과 동일한 `nvidia/nemotron-nano-12b-v2-vl:free`를 텍스트 전용으로 사용하려 했으나,
    실제 테스트 결과 한국어 장문(조리 순서) 생성 시 문장 중간에 한자/아랍어 등 무관한 문자가 섞이는
    품질 문제가 확인되어 텍스트 생성 전용 모델로 교체함.
  - 후보 비교(냉장고 재료: 토마토/오이/양배추/계란 기준): `nemotron-nano-9b-v2:free`도 동일한 문자 섞임 문제 발생,
    `gpt-oss-20b:free`와 `nemotron-3-super-120b-a12b:free`는 자연스러운 한국어 출력 확인 → 속도/안정성 기준으로 `gpt-oss-20b:free` 채택
- **입력**: PRD_01의 `ingredients: string[]` 결과를 그대로 활용
- **출력 형식**: 프롬프트에서 JSON 구조로 응답하도록 강제 (자유 서술형 텍스트는 파싱 어려움)
  - 예시 구조:
    ```json
    {
      "recipes": [
        {
          "name": "계란볶음밥",
          "usedIngredients": ["계란", "밥", "당근"],
          "missingIngredients": ["간장"],
          "steps": ["...", "..."],
          "estimatedTimeMinutes": 15
        }
      ]
    }
    ```
- **에러 처리**: PRD_01과 동일하게 free 모델의 502/504에 대비한 재시도 로직 필요
- **프롬프트 언어**: 한국어 응답 고정 (사용자 대상이 한국어 사용자이므로)

## API 인터페이스 (초안)
```
POST /api/recipes
Content-Type: application/json
Body: { "ingredients": ["계란", "우유", "당근"] }

Response 200:
{
  "recipes": [
    {
      "name": "string",
      "usedIngredients": ["string"],
      "missingIngredients": ["string"],
      "steps": ["string"],
      "estimatedTimeMinutes": 0
    }
  ]
}

Response 4xx/5xx:
{ "error": "메시지" }
```

## 성공 기준
- 식재료 3~10개 입력 시 실행 가능한 레시피 1개 이상 생성
- 모델 응답이 JSON 파싱 가능한 형태로 오지 않을 경우(형식 이탈) 이를 감지하고 에러 처리
- 부족한 재료가 있는 경우 이를 명확히 구분해 표시

## 미결 사항
- 레시피 개수를 늘리려면(1개 → 여러 개) 모델을 더 큰 모델로 교체하거나, 여러 개를 병렬로 1개씩 요청하는 방식이 필요함
- 알레르기/식단 제약(비건, 특정 재료 제외 등) 반영 여부 - 이후 단계에서 확장 가능하도록 프롬프트 구조는 열어둠
